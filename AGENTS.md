# AGENTS.md — PrepTrack

Guidance for AI agents (Claude, Cursor, Copilot, etc.) working on this codebase.

## Project Overview

PrepTrack is a UPSC preparation tracker built with React + TypeScript + Vite on the frontend and Supabase as the backend. The app is a PWA (installable, offline-capable).

## Key Commands

```bash
npm run dev          # start dev server (Vite)
npm run build        # TypeScript check + production build
npm run lint         # ESLint
npm run preview      # preview production build
```

## Tech Stack & Patterns

- **Supabase** — all DB queries go through `src/lib/supabase.ts`. Use the typed client. Auth is handled via Supabase Auth.
- **TanStack Query** — server state (fetching/mutating Supabase data). Avoid managing server state in Zustand.
- **Zustand** — client/UI state only (sidebar open, active filters, etc.). Stores live in `src/store/`.
- **React Hook Form + Zod** — all forms use RHF with Zod schemas for validation. No uncontrolled inputs.
- **TipTap** — rich text editor for notes. Extensions are registered in the editor component, don't install conflicting ones.
- **shadcn/ui** — component primitives in `src/components/ui/`. Don't modify these directly; extend via wrapper components.
- **Tailwind CSS** — utility-first. Use `cn()` from `src/lib/utils.ts` for conditional class merging.

## File Conventions

- Pages → `src/pages/` (one file per route)
- Shared components → `src/components/`
- Feature-specific components → colocate inside `src/components/<feature>/`
- Hooks → `src/hooks/` (prefix with `use`)
- Types → colocate with the file that owns them; shared types in `src/types/`

## Environment Variables

All env vars are prefixed `VITE_` and accessed via `import.meta.env`. Required vars:

```
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
```

Never commit `.env` files. Never log env vars.

## Important Notes

- The app targets UPSC aspirants — domain terms (Prelims, Mains, GS papers, Optional subject) are used throughout. Don't rename them to generic terms.
- Supabase RLS (Row Level Security) is enabled. All DB policies are user-scoped. When adding new tables, always add RLS policies.
- The PWA service worker is auto-generated by `vite-plugin-pwa`. Don't manually edit the generated `sw.js` in `dist/`.
- Sentry is initialized in `src/main.tsx`. Don't remove it; wrap risky operations in try/catch so errors surface.
- `react-pdf` requires a worker — it's configured in the PDF viewer component. Don't change the worker URL without testing.

## What NOT to do

- Don't use `any` type — fix the actual type instead.
- Don't bypass Supabase RLS by using the service role key in frontend code.
- Don't add large dependencies without checking if existing ones (TipTap, Recharts, date-fns) already cover the use case.
- Don't store sensitive data (tokens, keys) in Zustand or localStorage unencrypted.
